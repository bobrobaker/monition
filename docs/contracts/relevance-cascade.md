# Contract: relevance-cascade

Durable artifacts for the relevance-cascade filter (Phase 5). Owns interface semantics
for the label dataset, the trained head, and per-firing score logging. Buckets reference
the relevant section; they do not restate it.

Background: `docs/decisions/2026-06-18-noise-targets-the-filter-not-the-gate.md`,
*Update — 2026-06-21*. Spike: branch `spike/relevance-cascade`.

## 1. Label dataset (owned by B01)

A `(prompt, row) → label` table used to train and evaluate the relevance head.

- **unit**: one *firing* — a (prompt context, takeaway row) pair that fired `on_demand`.
- **fields**:
  - `takeaway_id` — the row that fired (FK to `takeaways.id`).
  - `prompt` — the firing's `trigger_context` (raw user prompt text; may be truncated).
  - `label` ∈ {`helpful`, `noise`} — human rating, OR `oracle_relevant` ∈ {0,1} when
    generated by the offline LLM oracle. The two sources MUST be kept distinguishable
    (a `label_source` ∈ {`human`, `oracle`} column) — never silently mixed, because the
    held-out test set must be **human-labeled only**.
  - `split` ∈ {`train`, `test`} — assigned by **whole-prompt grouping** (all firings
    sharing a normalized prompt go to the same split). Grouping key: prompt lowercased,
    whitespace-collapsed, truncated to 120 chars (matches the spike's leakage guard).
- **conservation**: every rated firing in the hub is either in the dataset or counted in
  a `skipped` tally with a reason (no `trigger_context`, unrated, etc.). Report the tally.
- **provenance**: regenerated from the hub, not hand-edited. Contains real session
  prompts → **never committed to git** (see spike `.gitignore`); lives under a gitignored
  data dir.

## 2. Trained head artifact (owned by B02)

The learned relevance head `L2′`, serialized for the runtime to load.

- **inputs at train/infer time**: L2-normalized prompt embedding ⊕ row embedding (the
  full vectors from `embed._embed_raw`, NOT cosine). Feature construction MUST match
  between training and inference — name it as an invariant.
- **artifact**: weights + the exact feature spec (model name/dim, normalization, feature
  layout, any PCA basis) + a `version` + the train/test AUC at creation. Format and
  on-disk location are an **open decision B02 must resolve and record here** (candidate:
  a versioned file under the managed cache `embed._weights_dir()`-sibling, NOT the Dolt
  store — weights are not row data).
- **embedding-version coupling**: the head is only valid for the embedding model it was
  trained against (`embed.MODEL_NAME`). Store that model id in the artifact; the runtime
  refuses to load a head whose model id ≠ the live `embed.MODEL_NAME`.
- **validity**: an artifact ships only if it clears the B02 usefulness gate on the
  human-only test split (operating-point chosen in B05, not at train time).

## 3. Per-firing score logging (decision in B01, wiring in B04, may touch firings schema)

To accumulate future training data, the runtime MAY log, per firing, the L2′ score and/or
the prompt embedding. This is an **addition to the firings contract** (`docs/contracts/
takeaway-store.md`) if pursued.

- **decision to make (B01)**: log nothing (regenerate from re-embedding later) vs. log the
  L2′ score (cheap, 1 float) vs. log the full prompt embedding (heavy, enables re-training
  without re-embedding). Default lean: log the scalar score; re-embed on demand.
- **if a firings column is added**: bump the firings contract, keep it nullable
  (back-compat), and remember Dolt omits NULL columns from JSON (`row.get`, not `row[]`).
- **anti-substitution**: a logged score is tied to the *head version* that produced it —
  log `head_version` alongside, or the score is uninterpretable after a retrain.
